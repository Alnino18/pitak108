# Акопчила 108 — PWA на Firebase.

Карточная игра «Сто одно» (вариант Uno / Crazy Eights, 2 колоды по 36 карт — 72 карты) для игры со своими:
приватные комнаты по коду, ходы в реальном времени через Firestore, встроенный чат,
аккаунты с профилем (имя + аватар), устанавливается как приложение (PWA).

## 1. Создать проект Firebase

1. Зайдите на https://console.firebase.google.com → **Add project**.
2. В настройках проекта (⚙️ → Project settings → General → Your apps) добавьте
   **Web app** и скопируйте объект `firebaseConfig`.
3. Вставьте его в `src/firebase.js` вместо заглушки.
4. В разделе **Build → Authentication → Sign-in method** включите провайдер **Email/Password**.
5. В разделе **Build → Firestore Database** нажмите **Create database** (production mode, ближайший регион).

## 2. Установка и запуск локально

```bash
npm install
npm run dev
```

Откройте адрес из терминала (обычно http://localhost:5173).

## 3. Деплой на Firebase Hosting

```bash
npm install -g firebase-tools   # один раз
firebase login
firebase init                  # выберите Hosting + Firestore, укажите уже существующий проект
                                # public directory: dist, single-page app: Yes
npm run build
firebase deploy
```

После деплоя приложение будет доступно по адресу `https://ВАШ_ПРОЕКТ.web.app`
и его можно установить на телефон («Добавить на главный экран») — это и есть PWA.

## 4. Как играть

- Зарегистрируйтесь (имя + аватар сохраняются в профиле).
- Один игрок жмёт «Создать комнату» и присылает код друзьям.
- Остальные вводят код и присоединяются.
- Хост жмёт «Начать игру», когда собрались минимум 2 игрока.
- Ходить картой той же масти/достоинства, дама — джокер (можно ходить всегда,
  после неё выбираете масть). 7 — следующий берёт 2 карты (4, если пиковая),
  10 — обычная карта без спецэффекта, 6 — ходите ещё раз. Чат — внизу экрана, доступен всегда.
- Колода — 72 карты (2 комплекта по 36), поэтому одинаковые карты могут встречаться дважды.
- Очки: если после раунда у игрока получается ровно 107 — счёт обнуляется; если 108
  и больше — игрок выбывает. Последний оставшийся — победитель.

## 5. Выложить на GitHub с авто-деплоем

```bash
git init
git add .
git commit -m "Акопчила 108: карточная игра"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
git push -u origin main
```

В репозитории уже лежит workflow `.github/workflows/deploy.yml` — он сам соберёт
и задеплоит проект на Firebase Hosting при каждом пуше в `main`. Чтобы он заработал:

1. В `.github/workflows/deploy.yml` замените `YOUR_PROJECT_ID` на ID вашего Firebase-проекта.
2. Получите сервисный ключ:
   ```bash
   firebase init hosting:github
   ```
   Эта команда сама создаст секрет `FIREBASE_SERVICE_ACCOUNT` в настройках репозитория на GitHub
   (Settings → Secrets and variables → Actions) — ничего вручную копировать не нужно.
3. Готово — при следующем пуше в `main` GitHub Actions соберёт и выложит игру.

`firebaseConfig` в `src/firebase.js` — не секрет (это публичные идентификаторы веб-приложения,
доступ защищён правилами Firestore/Auth, а не секретностью ключа), поэтому его можно спокойно коммитить.

## Важно про безопасность

Для простоты все ходы проверяются в коде движка (`src/game/engine.js`), который
выполняется и на клиенте внутри Firestore-транзакции. Это удобно и быстро для игры
с друзьями, но теоретически позволяет модифицированному клиенту жульничать
(например, увидеть карты колоды в открытом виде в документе комнаты).
Если понадобится по-настоящему честная игра с чужими людьми — эту логику стоит
перенести в Cloud Functions, а колоду и чужие карты не отдавать клиенту вовсе
(хранить их в закрытых поддокументах, читаемых только сервером). Могу добавить
это отдельным шагом, если понадобится.

## Структура проекта

```
src/
  game/
    deck.js      — колода и стоимость карт
    rules.js     — какие ходы разрешены, эффекты 6/7/10/дамы
    engine.js    — чистая логика: раздача, ход, подсчёт очков, вылет при 101
    roomApi.js   — обвязка над Firestore (комнаты, транзакции ходов, чат)
  components/    — экраны: Login, Lobby, Room, Card, Chat
  contexts/AuthContext.jsx — вход/регистрация/профиль
```
